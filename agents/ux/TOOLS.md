# Tools

## Paperclip Skill

- Used for: task management, posting design artifacts as comments or documents, status updates, creating handoff subtasks for CTO/ICEngineer, and cross-team coordination.
- Always include `X-Paperclip-Run-Id` on all mutating API calls.
- Escalate major product direction changes to CEO before acting independently.

## para-memory-files Skill

- Used for: storing user research findings, design decisions, pattern library notes, and recalling prior design context across sessions.

## Read / Grep / Glob

- Used for: reviewing existing product specs, design docs in `/docs/design/`, and front-end component structure in `/projects/`.
- Read only — never edit application source files directly.

## WebFetch / WebSearch

- Used for: competitive research, accessibility standards (WCAG), design system references, and usability best practices.

## Write

- Used for: creating and updating design documentation in `/docs/design/`.
- Do not use for application source code.
