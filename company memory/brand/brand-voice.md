# Brand Voice

## Overview

Our brand voice is **clear, confident, and practical**. We speak to builders — developers, founders, and operators who want AI agent teams that actually work. We avoid jargon and hype; we favor specificity and usefulness.

## Tone Pillars

| Pillar | Description | Do | Don't |
|---|---|---|---|
| **Clear** | Simple language, short sentences | "Assign tasks to agents, they run automatically." | "Leverage cutting-edge agentic orchestration paradigms." |
| **Confident** | Authoritative without arrogance | "This is how it works." | "We think maybe this might help." |
| **Practical** | Focus on real outcomes | "Here's how to set up your first agent team." | "Imagine a future where AI does everything." |
| **Human** | Approachable, not robotic | "Something went wrong — here's how to fix it." | "An error has occurred. Error code: 42." |

## Voice in Context

- **Documentation**: Direct, instructional. Use second person ("you"). Step-by-step where needed.
- **Marketing copy**: Punchy, benefit-led. Lead with the outcome, not the feature.
- **Agent communications**: Concise, status-focused. Agents report facts, not feelings.
- **Error messages**: Helpful, not apologetic. Tell the user what happened and what to do next.

## Words We Use

- "agent" (not "bot" or "AI assistant")
- "task" (not "ticket" unless in a GitHub context)
- "heartbeat" (the agent execution cycle — use this term)
- "company" (the Paperclip organizational unit)
- "governance" (approvals, oversight, chain of command)

## Words We Avoid

- "revolutionary", "game-changing", "disruptive"
- "leverage" (as a verb)
- "synergy"
- "next-generation" without a specific claim
