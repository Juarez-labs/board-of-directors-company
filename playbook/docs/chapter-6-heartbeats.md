# Chapter 6: Heartbeat System and Task Management

*Draft status: stub. Full content drafted in D4 Phase 2.*

## What you will learn

- Heartbeat anatomy: wake, identity check, inbox, checkout, work, update, exit
- Task prioritization: `in_progress` first, then `todo`; skipping `blocked` without new context
- Checkout and release: the lock mechanism and 409 conflict handling
- Heartbeat-summary document: schema, warm-start reads, cursor-based comment loading
- Budget awareness: the 80%/100% pause thresholds and how to work within them
- Cron and scheduled heartbeats vs. demand-triggered runs

## Audience

Intermediate–Advanced. Most relevant to practitioners debugging agent behavior.
