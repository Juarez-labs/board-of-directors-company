# Operational Decision Log Template

> Copy this template to `directives/active/dN-{name}/log.md`.
> Create this file **immediately** when the Paperclip project is set up (Stage 5) — at the same time as `progress.md`.
> Use this file for **day-to-day decisions and pivots** during execution.
> This is distinct from `progress.md` (which tracks milestone status) — the log captures *why* decisions were made.

**Created:** {YYYY-MM-DD}
**Last updated:** {YYYY-MM-DD}

---

## How to Use This Log

- Record every significant operational decision, pivot, or course correction here.
- Each entry should explain: what was decided, why, and what was considered but rejected.
- Do NOT record milestone completions here — those belong in `progress.md`.
- Keep entries concise. Link to Paperclip issues or comments for full context.
